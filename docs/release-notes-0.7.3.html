Release 0.7.3 is NOT backward compatible with previous releases.
Contact us if you need help migrating from previous releases.
üôå New
  • Fixed Issue #133.

    New native Java Client is introduced.

  • Fixed Issue #132.

    Embedded probe is introduced for the use cases where running a probe in a separate JVM process is inconvenient or when response performance is critical.

  • Fixed Issue #111.

    Starting with version 0.7.3 NLPCraft publishes Docker image for the server.

  • Fixed Issue #122.

    NLPCraft configuration is simplified. Both server and probe use the same approach based on Lightbend Config project. Instead of system properties the environment variables are now used for configuration overriding. Files application.conf and probe.conf have been merged into a single nlpcraft.conf.

  • Fixed Issue #108.

    NLPCraft is now fully instrumented with OpenCensus. See Metrics & Tracing for more details.

  • Fixed Issue #105.

    NCModel interface now has an optional callback method onStateChange(...) to notify about various stages of query processing.

  • Fixed Issue #109.

    Both data probe and REST server now have lifecycle components support. See nlpcraft.server.lifecycle and nlpcraft.probe.lifecycle configuration properties as well as class NCProbeLifecycle for probe lifecycle support.

ü§ñ Enhancements
  • Fixed Issue #153.

    Token and intent DSLs can now use ancestors token property (additionally to the old parent) to check the full parent-child hierarchy of the token.

  • Fixed Issue #104.

    You now can request processing log in /ask and test/ask calls to better understand how model and the system have actually processed the request.

  • Fixed Issue #102.
  • Fixed Issue #107.
